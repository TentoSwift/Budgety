# 0001_purchase-double-buy-guard — やりとり要点

> 生トランスクリプトはコピペしない。後から検索したい要点だけ。

## 効いた指示 / 決定
- 「買い切りプランを買ったあとにサブスクにも入れてしまう？」→ 調査。
- アプリ内は `isPremium` で購入 UI を隠すので不可。ただし `purchase()` 自体は無防備。
- 「サブスクリプショングループに買い切りも含めれる？」→ **不可**（非消耗型は対象外）。
- 結論: 相互排他はアプリ内ゲートで担保するのが唯一の正解 → モデル層に1行ガード追加。

## 方針転換
- 当初「買い切り後にサブスク併用」リスクを高めに説明したが、最初の購入で
  `isPremium` が立ち全購入 UI が消えるため、正規フローでの実リスクは低いと訂正。

## ハマった点と解決
- 作業中、Bash の cwd がセッションのプロジェクトルート (`~/Downloads/Claud`) に
  drift し、ホーム直下の空 git リポジトリ (`~/.git`) を指していた。
  → git 操作は `git -C <repo>` か `cd <repo> &&` で明示。`~/.git` は削除済み。

## 学び（→ MEMORY.md 反映候補）
- 非消耗型はサブスクリプショングループに入れられない＝買い切り×サブスクの排他は
  Apple ネイティブ不可。アプリ内ゲート + `purchase()` ガードで担保。
  → memory: budgety_purchase_design.md
